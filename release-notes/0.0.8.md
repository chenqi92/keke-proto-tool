# Version 0.0.8

## 🚀 主要新功能 | Major New Features

### 🔗 高级连接管理 | Advanced Connection Management
- **连接超时配置**: 可配置连接超时时间（5-300秒，默认30秒）
  - **Connection Timeout Configuration**: Configurable timeout settings (5-300 seconds, default: 30s)
- **智能自动重连**: 支持指数退避算法的自动重连机制
  - **Intelligent Auto-Reconnect**: Automatic reconnection with exponential backoff retry logic
  - 可配置重试次数（1-10次）| Configurable retry attempts (1-10 times)
  - 可配置初始延迟（1-30秒）| Configurable initial delay (1-30 seconds)
  - 最大重试延迟限制为30秒 | Maximum retry delay capped at 30 seconds
- **实时连接状态**: 增强的状态指示器，显示详细连接状态信息
  - **Real-time Connection Status**: Enhanced status indicators with detailed connection state information

### 📡 自动数据传输 | Automatic Data Transmission
- **循环数据发送**: 客户端会话支持自动周期性数据传输
  - **Cyclic Data Sending**: Automatic periodic data transmission for client sessions
- **灵活间隔设置**: 可配置发送间隔（100毫秒到1小时）
  - **Flexible Intervals**: Configurable sending intervals (100ms to 1 hour)
- **多种数据格式**: 支持文本、十六进制、二进制和JSON格式
  - **Multiple Data Formats**: Support for Text, Hex, Binary, and JSON formats
- **实时统计**: 实时跟踪已发送消息和错误计数
  - **Real-time Statistics**: Live tracking of sent messages and error counts
- **手动控制**: 支持启动/停止自动发送，带有可视化指示器
  - **Manual Control**: Start/stop automatic sending with visual indicators

## 🎨 用户界面改进 | UI/UX Improvements

### 📋 连接管理面板 | Connection Management Panel
- **可折叠界面**: 连接管理设置默认隐藏，减少界面混乱
  - **Collapsible Interface**: Connection management settings hidden by default to reduce UI clutter
- **右对齐切换按钮**: "连接管理"按钮位于数据格式行的最右侧
  - **Right-aligned Toggle Button**: "连接管理" button positioned on the far right of the data format row
- **直接展开**: 简化的面板直接展开，无中间状态
  - **Direct Expansion**: Streamlined panel that expands directly without intermediate states
- **紧凑布局**: 优化间距和填充，更好地利用空间
  - **Compact Layout**: Optimized spacing and padding for better space utilization

### 📊 增强的会话列表和工作区 | Enhanced Session List & Workspace
- **彩色状态徽章**: 不同连接状态的可视化状态指示器
  - **Colored Status Badges**: Visual status indicators with color-coded badges
  - 🟢 已连接（绿色）| Connected (Green)
  - 🟡 连接中/重连中（黄色）| Connecting/Reconnecting (Yellow)
  - 🔴 已断开/错误（红色）| Disconnected/Error (Red)
- **自动展开树导航**: 默认展开所有工作区树节点，提高可见性
  - **Auto-expanded Tree Navigation**: All workspace tree nodes expanded by default for better visibility
- **内联连接编辑**: 在会话操作区域直接编辑IP地址和端口
  - **Inline Connection Editing**: Direct editing of IP addresses and ports in session operation areas

### ✨ 简化的会话创建 | Streamlined Session Creation
- **简化新会话对话框**: 从初始会话创建中移除复杂的连接管理设置
  - **Simplified New Session Dialog**: Removed complex connection management settings from initial session creation
- **专注基本要素**: 会话创建现在专注于基本连接参数
  - **Focus on Essentials**: Session creation now focuses on basic connection parameters
- **创建后配置**: 会话创建后可通过专用连接管理面板进行高级配置
  - **Post-creation Configuration**: Advanced connection management available after session creation

## 🔧 技术改进 | Technical Improvements

### 🏗️ 服务架构 | Service Architecture
- **ConnectionManagerService**: 集中式连接生命周期管理，具有适当的错误处理
  - **ConnectionManagerService**: Centralized connection lifecycle management with proper error handling
- **AutoSendService**: 专用的自动数据传输服务，具有定时器管理
  - **AutoSendService**: Dedicated service for automatic data transmission with timer management
- **服务注册模式**: 清晰的关注点分离，适当的服务注册和清理
  - **Service Registry Pattern**: Clean separation of concerns with proper service registration and cleanup

### 🛡️ 类型安全和验证 | Type Safety & Validation
- **增强的TypeScript类型**: 改进连接状态和配置的类型定义
  - **Enhanced TypeScript Types**: Improved type definitions for connection states and configurations
- **运行时验证**: 网络配置参数的客户端验证
  - **Runtime Validation**: Client-side validation for network configuration parameters
- **向后兼容性**: 保持与现有会话配置的兼容性
  - **Backward Compatibility**: Maintained compatibility with existing session configurations

## 🐛 错误修复 | Bug Fixes

### 🏷️ StatusTag组件修复 | StatusTag Component Fixes
- **修复运行时错误**: 解决了"Cannot read properties of undefined (reading 'icon')"错误
  - **Fixed Runtime Error**: Resolved "Cannot read properties of undefined (reading 'icon')" error
- **增强状态处理**: 强大处理来自Rust后端的字符串和对象状态值
  - **Enhanced Status Handling**: Robust handling of both string and object status values from Rust backend
- **Rust枚举兼容性**: 正确规范化Rust枚举变体
  - **Rust Enum Compatibility**: Proper normalization of Rust enum variants
- **回退机制**: 对未知状态类型优雅回退到'disconnected'状态
  - **Fallback Mechanisms**: Graceful fallback to 'disconnected' state for unknown status types

### 🎯 UI一致性 | UI Consistency
- **跨协议一致性**: TCP、UDP和WebSocket会话的统一连接管理界面
  - **Cross-protocol Consistency**: Uniform connection management interfaces across TCP, UDP, and WebSocket sessions
- **视觉对齐**: 连接管理按钮和状态指示器的正确对齐
  - **Visual Alignment**: Proper alignment of connection management buttons and status indicators
