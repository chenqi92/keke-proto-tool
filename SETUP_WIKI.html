❓ 为什么还显示 "Create the first page"？

原因：GitHub Wiki 是一个独立的 Git 仓库，需要单独激活。

我们创建的文件在主仓库的 wiki/ 文件夹中，但 GitHub Wiki 需要推送到独立的 Wiki 仓库。

🚀 第 1 步：启用 Wiki 功能

  1. 点击下面的按钮访问仓库设置
  2. 滚动到 "Features" 部分
  3. 勾选 ✅ "Wikis" 复选框
  4. 保存设置
打开仓库设置 →

📝 第 2 步：创建第一个页面

方法 A：复制粘贴（推荐）

  1. 点击下面的按钮访问 Wiki
  2. 点击 "Create the first page" 按钮
  3. 标题输入：Home
  4. 打开项目中的 wiki/Home.md 文件
  5. 复制全部内容
  6. 粘贴到 GitHub Wiki 编辑器
  7. 点击 "Save Page" 按钮
打开 Wiki → 查看 Home.md

方法 B：简单内容（快速）

如果只是想快速激活 Wiki，可以先输入简单内容：

  • 标题：Home
  • 内容：Wiki is being set up...

然后保存，稍后通过 Git 推送完整内容。

🔄 第 3 步：推送所有页面

创建第一个页面后，运行以下命令推送所有 13 个页面：

Windows (PowerShell):

cd .wiki-temp git remote set-url origin git@github.com:chenqi92/keke-proto-tool.wiki.git git push -u origin master --force cd .. Remove-Item .wiki-temp -Recurse -Force

或使用自动化脚本:

.\scripts\publish-wiki-after-enable.ps1

Linux/macOS:

cd .wiki-temp git remote set-url origin git@github.com:chenqi92/keke-proto-tool.wiki.git git push -u origin master --force cd .. rm -rf .wiki-temp
